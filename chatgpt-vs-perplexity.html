    Perplexity AI 与 ChatGPT：模型比较

    Perplexity AI 和 ChatGPT 都是基于大型语言模型 (LLM) 构建的，但也存在一些关键区别：

    • ChatGPT 由 OpenAI 开发，在多模态交互 (LMM) 和创意工作方面表现更出色。它基于 GPT‑4o / GPT‑4.5，支持文本、图像和音频的输入和输出。此外，它还提供混合媒体创作、语音聊天和图像编辑等更丰富的功能，使其成为一款适合广泛用户的全面工具。
    • Perplexity 由 Perplexity Inc. 开发，采用其自主研发的检索增强系统 (RAG)，在准确性和引用透明性方面具有优势。每个答案都包含可点击的来源，非常适合学生、研究人员、市场分析师或任何需要高度可信且可追溯信息的人士。

    💬 什么是 LLM 和 LMM？

    • LLM(大型语言模型) 是当今最常见的人工智能类型。它能够阅读、书写和理解语言，以文本作为其主要的输入和输出形式。
    • LMM(大型多模态模型) 是 LLM 的自然演化。除了文本之外，它还可以处理图像、视频、音频、代码和其他格式。这是迈向更通用的人工智能 (AGI) 的重要一步。

    大多数 ChatGPT 用户所依赖的 GPT‑4o 模型是一个经典的 LLM(同时也是一个 LMM)，因此它可以处理多模态输入(图像、音频、文本)。而 Perplexity AI 主要基于 LLM 构建，其多模态交互能力较弱，更倾向于文本理解 + 实时检索 + 来源引用。

    那么，这些人工智能模型目前的评级如何？在ArtificialAnalysis.ai网站上，他们的"人工智能指数"对比图衡量了领域知识、推理、数学和编程等领域的表现。

    从结果来看，OpenAI 的 GPT-4o Pro(预估)和官方 GPT-4o (O3) 分别以 71 分和 70 分位居榜首，表明它们在整体性能上仍然领先。Claude 3.5 Sonar 系列的得分在 43 到 49 分之间，略低于 GPT-4.5 的 53 分。

    GPT-4o 的排名相对较低，但对于一个同时处理文本和图像的多模态模型来说，它的语言理解能力仍然相当出色。毕竟，它必须在推理能力和多模态性之间取得平衡——鱼与熊掌不可兼得。

    由于它们基于不同的型号，定位也不同，使用起来感觉也有所不同。下面，我将逐一分析它们的优缺点。

    我如何测试它们？

    为了确保测试结果客观一致，我在两台相同的 Mektec 便携式电脑上对 ChatGPT 4o 和 Perplexity Pro 进行了评测，每台电脑均配备 AMD Ryzen 7 7840HS 处理器、Radeon 780M 集成显卡、16GB RAM 和 1TB SSD。两台设备均在相同的网络条件下进行测试。

    我们的评估重点关注两种人工智能工具的理解能力和输出质量。对于开发相关的任务，我们测试了它们生成功能代码的能力。对于创造性任务，我们评估了它们写作的流畅性和连贯性——例如，要求它们分享对"失去与希望"的看法。对于基于搜索的任务，我们考察了它们是否能够提供最新信息以及可靠的来源，例如检索有关"50501运动"的详细信息，并比较每个人工智能处理引用的方式。

    我还测试了它们的数据处理能力，例如生成带有回归线的散点图、图像生成和分析，以及多轮对话中的上下文理解和响应一致性——所有这些都旨在尽可能地模拟现实世界的使用情况。

    ChatGPT 的全面性能

    作为当今人工智能领域的领军人物之一，ChatGPT 的最大优势在于其全面均衡的性能。无论是基础语言理解、情感交互、编程、视觉推理，还是图像和音频生成，ChatGPT 都取得了业界领先的成绩。

    虽然在某些特定任务中，ChatGPT 的精度可能无法达到某些领域特定模型的极限，但它的各种功能能够无缝协作，没有明显的弱点。简而言之，ChatGPT 是人工智能领域真正的全能型人才。

    数据分析与整合技能

    在我提交之前收集的学生学习时间和学业成绩数据后，ChatGPT 在几秒钟内对其进行了分析，生成了散点图并拟合了回归线——所有这些都具有令人印象深刻的速度和准确性。

    图表清晰地显示了学习时间与学业成绩之间的高度正相关关系。GPT 生成的输出简洁明了，结构清晰，准确反映了原始数据分布。拟合的回归线有效地模拟了总体趋势，既具有可解释性，又具有预测价值。

    编码能力

    根据 JetBrains 2024 年开发者生态系统报告，现在几乎一半的开发者定期使用 ChatGPT 执行编码任务——尤其是生成代码片段、调试和学习实现逻辑。

    虽然作为一名科技博主，我对编程有一些基本的了解，但我并非专业开发者。ChatGPT 真正强大的地方在于，它能让像我这样的非专业人士轻松构建和部署应用或游戏。例如，我使用 ChatGPT o4-mini-high 模型创建了一个简单的炸弹人风格游戏——无需本地设置，整个过程流畅且对初学者友好。

    我所做的只是输入一个简单的提示："你能写一个经典的炸弹人风格的游戏并让我预览一下吗？使用任何你能找到的图像作为视觉效果。"

    它很快就给出了令人满意的结果。更让我印象深刻的是OpenAI最近推出的"思维可视化"功能，它让我能够实时看到它的逐步推理过程。这种感觉真的像是在与一位非常聪明的伙伴合作。

    在这个任务中，ChatGPT 最初考虑使用 Python 来实现，但很快意识到我当前的环境不支持 Pygame 渲染。随后，它探索了其他解决方案，建议使用 HTML/JavaScript Canvas 或 React 进行交互。最终，它决定采用基于 Canvas API 的 HTML 和 JavaScript 解决方案——这对于基于 Web 的执行来说是一个明智且适应性强的选择。

    更重要的是，尽管我没有给出具体的视觉效果指示，ChatGPT 也展现出了令人印象深刻的适应性。一旦它意识到无法提取外部图像，它就会迅速转向使用简单的彩色块来表示游戏元素——这是一种优雅而实用的解决方法。

    后来，我觉得视觉效果可以稍微改进一下，所以我要求它更改某些元素的颜色。我的指令很模糊——几乎说不清楚——但 ChatGPT 仍然准确理解了我的意思，并相应地调整了视觉效果。

    Perplexity AI 在这方面的灵活性差得多。我发送了完全相同的提示，但它忽略了我要求预览结果的请求，也没有意识到我的本地环境不支持 Pygame。相反，它只是返回了一段 Python 代码，而没有考虑这段代码是否真的能在我的终端上运行。

    Perplexity AI 与 ChatGPT：语言理解比较

    虽然 Perplexity AI 无疑是实时搜索领域最强大的 AI 之一，但仅凭这一点还不足以评估一款 AI 工具。语言理解在整体用户体验中也扮演着同样重要的角色。

    Perplexity AI 确实具备对话助手的功能。它提供了与 ChatGPT 类似的聊天界面和交互风格。尤其是在其专业搜索模式下，它可以处理多轮对话、跟进问题，并分析用户意图以优化其响应。

    然而，在语境理解和情感互动方面，Perplexity 与 ChatGPT 相比略有不足。例如，当我要求它分享对"失去与希望"的看法时，它的回应显得过于机械化——缺乏情感深度，也缺乏那种能够真正进行情感交流的人工智能所应具备的人类般的同理心。

    相比之下，ChatGPT 对同一问题的回答——它对"失落与希望"的解读——则显得更加人性化。它不仅提供了一个情感丰富的答案，还触及了一层哲学思考。

    ChatGPT 能够以一种既真挚又深思熟虑的方式探索这些复杂的情感，展现出对情感的细致理解以及对生命意义的深刻思考。最终，回复温暖、富有洞察力，并真正引人入胜。

    Perplexity AI 更倾向于提供精确、结构化的信息响应，但它缺乏 ChatGPT 对话风格的自然流畅和情感细微差别。

    虽然 Perplexity 也能处理写作、文档和图像分析，甚至专业推理等任务，但我们的测试表明，它在灵活性和多功能性方面不如 ChatGPT。它并非这类任务的理想选择。

    常问问题

    Perplexity AI 比 ChatGPT 更好吗？

    Perplexity AI 擅长实时网页搜索和来源引用，是获取最新信息的理想之选。然而，对于大多数用户来说，在创意写作、推理、编程以及整体语言能力方面，ChatGPT 仍然更强大、功能更丰富。

    Perplexity AI 使用 GPT-4 吗？

    是的，Perplexity AI 为 Pro 用户提供 GPT-4 访问权限。默认情况下，它使用自己的模型进行搜索和合成，但 Pro 订阅用户可以根据任务选择 GPT-4、Claude 3 或 Mistral 等模型来生成响应。

    Perplexity AI 完全免费吗？

    Perplexity AI 提供免费版本，可使用其核心功能和网页搜索功能。不过，GPT-4、Claude 3 等高级模型和其他专业版功能需要订阅。因此，虽然免费使用，但完整访问权限并非完全免费。
